## Acerca del Proyecto

### Objetivos

El principal objetivo de **InventoMate** es el de impulsar el crecimiento de las ventas del cliente mientras se minimizan las pérdidas de recursos, como dinero y tiempo.

Para alcanzar nuestro objetivo nos centramos en:
* Ayudar al cliente a optimizar la gestión del inventario de la empresa.
* Otorgar al cliente la información necesaria para realizar una toma de decisiones estratégicas e informadas.
* Lograr que el cliente se mantenga competitivo en el mercado.
* Ayudar al cliente a satisfacer las demandas cambiantes del mercado.

## Modelos de Datos

### Modelo Entidad-Relación

En la base de datos relacional (MySQL) persistimos toda la información del sistema.

![Modelo ER](https://github.com/user-attachments/assets/c20e0ee5-da8d-482a-b34a-b395d835991d)

### Modelo de Datos No Relacional

Además del modelo relacional, **InventoMate** utiliza una base de datos no relacional (MongoDB). Esta base de datos se utiliza para almacenar los informes del cliente, lo que permite una mayor flexibilidad en la estructura de los datos.

![Diagrama de Documentos MongoDB](https://github.com/user-attachments/assets/d0f08653-2c96-4a54-9488-c41baa786b70)

Esta combinación de bases de datos relacional y no relacional permite gestionar los datos de manera eficiente y flexible, aprovechando las fortalezas de ambos tipos de bases de datos para cumplir con los requisitos del sistema.

## Contribuciones Personales

Durante el desarrollo de **InventoMate**, participé en las siguientes áreas:

* **Definición y Planificación del Proyecto:**
  - **Idea y Requisitos:** Contribuí a la conceptualización del proyecto y a la definición de requisitos funcionales y no funcionales.
  - **Gestión de Proyectos:** Colaboré en la planificación de tiempos de entrega y en la organización de tareas, asegurando una ejecución eficiente.

* **Diseño y Desarrollo:**
  - **Modelo de Datos:** Diseñé los modelos de datos relacionales (SQL) y no relacionales (NoSQL) para una integración efectiva.
  - **Arquitectura del Sistema:** Desarrollé el modelo de clases y la arquitectura general, estableciendo una base sólida para el sistema.
  - **API y Lógica de Negocio:** Implementé la API y la lógica de negocio con Spring Boot, creando una arquitectura robusta y escalable.
  - **Pruebas de API:** Realicé pruebas exhaustivas de la API en Postman para garantizar su funcionalidad y rendimiento.

* **Documentación y Comunicación:**
  - **Documentación del Proyecto:** Contribuí en el desarrollo de la documentación.
  - **Propuesta de Venta:** Contribuí a la propuesta de venta, destacando las características y beneficios del sistema.

Todas mis contribuciones al código están disponibles en el siguiente enlace, [backend_spring](https://github.com/SebaJuarez/inventomate/tree/main/backend/backend_spring)

### Soluciones Destacadas

* **Integración de Servicios:**
  - Implementé la comunicación entre el servicio de generación de informes y la API de Mercado Libre, integrando datos históricos del cliente para alimentar el modelo predictivo, mejorando la sincronización y el flujo de trabajo.

* **Base de Datos Híbrida:**
  - Logré una integración eficiente entre MySQL y MongoDB, optimizando el manejo y recuperación de datos.

* **Análisis de Datos y Recomendaciones:**
  - Desarrollé un sistema para procesar y analizar datos históricos del cliente, generando recomendaciones basadas en las tendencias del mercado mediante la API de Mercado Libre.

* **Servicios de Notificaciones y Seguridad:**
  - Diseñé e implementé servicios de notificaciones personalizados con Thymeleaf y EmailSender, y aseguré la protección de datos integrando Auth0 y Spring Security.

* **Informes de Auditoría:**
  - Creé informes de auditoría para rastrear el rendimiento del sistema, incluyendo tiempos de procesamiento, encuestas de satisfacción y seguimiento de errores, facilitando mejoras continuas.
